# Cold Start 문제와 해결 전략

**Summary**: 초기 데이터가 없는 상태에서 서비스를 시작해야 하는 문제. DAZZ는 뮤지션 셀프온보딩으로 해결한다.

## 문제 정의

DAZZ의 핵심 가치(관계도, 인사이트)는 **뮤지션 데이터가 있어야 작동**한다.
그런데 서비스 오픈 시점엔 데이터가 0이다 → 닭이 먼저냐 달걀이 먼저냐.

### 구체적인 콜드스타트 항목

| 항목 | 문제 |
| --- | --- |
| 도슨트 노트 | 창업자가 매주 작성하는 건 현실적으로 불가능 |
| 뮤지션 프로필 | 누가 처음 만들어주는가? |
| 협업 관계도 | 뮤지션 정보 없이 그래프 불가능 |
| 공연 정보 | 수기 입력은 지속 불가능 |

---

## 검토한 해결 옵션 3가지

### Option A — 공연 캘린더 퍼스트
재즈클럽 웹사이트 스크래핑으로 공연 정보부터 채운다.
- **장점**: 즉시 사용 가능한 데이터
- **탈락 이유**: 재즈피플과 차별점 없음. "Insight Navigator" 컨셉 사라짐

### Option B — 뮤지션 포트폴리오 퍼스트 ✅ 채택
창업자의 서울예대 재즈 인맥을 활용해 뮤지션 30~50명을 직접 온보딩.
뮤지션이 자기 프로필/공연이력/협업자를 직접 입력한다.
- **장점**: 컨셉 핵심(관계도, 디지털 이력서) 직접 구현. 뮤지션에게도 가치 있음
- **채택 이유**: 재즈 뮤지션들은 인스타그램에 공연 홍보글을 자체 작성하는 습관이 있어 데이터 입력 부담이 낮다

### Option C — 도슨트 노트 퍼스트
창업자가 직접 15개의 깊은 도슨트 노트를 작성.
- **탈락 이유**: 창업자가 지속적으로 글을 쓸 자신이 없음

---

## 채택 전략 상세 (Option B)

### 온보딩 방법
1. 완성된 프론트엔드 링크를 지인 뮤지션들에게 공유
2. "여기서 너를 홍보할 수 있어" 방식으로 자발적 참여 유도
3. 공연이력이 너무 오래된 경우 최근 N개월 데이터만 요청

### 인맥풀 활용 가능 범위
- ✅ "프로필 사진이랑 이름/악기 보내줘" → 가능
- ✅ "이 분이랑 같이 연주했지? 연결해도 돼?" → 가능 (1줄 카톡)
- ❌ "2019년부터 공연 이력 전부 정리해서 보내줘" → 아무도 안 함

### 초기 데이터 목표
- 뮤지션 30~50명
- 협업 관계 100건
- → 이 정도면 관계도로서 가치가 생긴다

---

## 결정이 미치는 영향

- **MVP 기능 우선순위**: 뮤지션 프로필 + 공연이력 셀프입력 UI가 P0
- **도슨트 노트**: MVP에서 미구현. 인터페이스만 정의 (Phase 2에서 Verified User가 작성)
- **크롤링 전략**: 재즈피플 크롤링은 법적 리스크 + 차별화 파괴로 채택 안 함
